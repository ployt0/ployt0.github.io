Ubuntu Setup

I want to share how I like to setup my Ubuntu Desktop. I've been away from it for too long and at the very least these notes should help me. The installation I'm running is Ubuntu 20.04. I like things minimal so there were just a handful of applications showing under "Show applications". (Please note not every app with an icon is displayed under this.) It appears the full-fat installation has things I might want and you might have already.

System-Monitor

gnome-shell-system-monitor-applet is the full name for what many simply refer to as "system-monitor". When I search the latter in "Ubuntu Software" I see it is already installed! I cannot find it in the graphical "Show Applications"->All. It is a doppelganger.

So paradoxxxzero's is definitely it. How then do I obey myriad conflicting instructions and get it running with the least fluff possible? Following the official README seems a fair bet. So it is more accurately called, "gnome-system-monitor". It was updated a few months ago in late 2021. The maintainer still cares. 

$ sudo apt install gir1.2-gtop-2.0 gir1.2-nm-1.0 gir1.2-clutter-1.0 gnome-system-monitor

It sells it to me at just 70kB.

Hmm, not seeing anything in "Show Applications", nothing in the setting panel either.

That was just the pre-requisites. Now I'm slightly confused by what follows:

  • Browser installation
  • Repository installation
  • Manual installation

I was disappointed to learn, "browser installation", lives up to its name. It is offering Click here to install browser extension. Altering the "gnome shell status bar" is not my understanding of what a browser extension can do. I've never followed this link before in the many Ubuntu installs I've done.

Stack Overflow to the rescue. I feel I've trodden this path before and been a little peeved that I installed pre-requisites (however small) for something I did not want nor need (despite its uncanny name). In his SO answer, Tully writes to:

$ sudo apt-get install gnome-shell-extension-system-monitor

That used just 300kB more disk space. Nothing showing just yet but Tully advises choosing a path from:

  • Log out/in
  • Reboot
  • Restart gnome-session

I used gnome-extensions enable system-monitor@paradoxxx.zero.gmail.com from another answer in my efforts to avoid rebooting. Tully's was shorter but left some room for confusion over what he meant by restarting the gnome session.

It works after much patience with: sudo systemctl restart display

I've since discovered other gnome shell extensions that require gnome session restarts. Pressing Alt+F2, and entering "r" will do it. Eventually. This does cause programs to close.
